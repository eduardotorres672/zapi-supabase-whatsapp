# Contribuindo

Contribuições são bem-vindas — desde correções simples até novas funcionalidades.

## Como contribuir

1. Faça um fork do repositório
2. Crie uma branch descritiva: `git checkout -b feat/nome-da-feature`
3. Faça suas alterações e commit: `git commit -m "feat: descrição clara do que foi feito"`
4. Abra um Pull Request explicando o que foi alterado e por quê

## Padrões do projeto

- Código em português (variáveis, logs, comentários) para manter consistência com o projeto original
- Mensagens de commit em inglês seguindo [Conventional Commits](https://www.conventionalcommits.org/)
- Sem dependências novas sem justificativa no PR

## Ideias de contribuição

- Suporte a envio de imagens ou arquivos via Z-API
- Relatório de envios exportado em CSV
- Interface CLI interativa com seleção de template
- Testes automatizados com `pytest` e mock da Z-API
